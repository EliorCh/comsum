MAC ו-ARP

‎48 ביטים פיזיים שמוטבעים בכרטיס במפעל · ARP ממפה IP ל-MAC.

כל כרטיס רשת בעולם יש לו כתובת MAC ייחודית בת 48 ביטים — מוטבעת במפעל ולא משתנה. ‎MAC משמש להעברה פיזית בין שכנים, לעומת IP שזה לוגי וגלובלי. כשמחשב צריך לשלוח לשכן ב-LAN — הוא מכיר את ה-IP, אבל לא את ה-MAC. ARP פותר את זה.

כתובת MAC — מבנה

📘‎48 ביטים, שני חלקים
  • אורך: 48 ביטים = 6 בתים.
  • כתיב: בהקסה, מופרד בקווים: 00:1A:2B:3C:4D:5E.
  • ‎3 בתים ראשונים (24 ביט): מזהה יצרן (OUI - Organizationally Unique Identifier).
  • ‎3 בתים אחרונים (24 ביט): מספר סידורי של הכרטיס אצל היצרן.

כל יצרן (Intel, Apple וכו') קונה OUI ייחודי מ-IEEE, ומקצה לו ‎2²⁴ ≈ 16 מיליון כתובות. אם נגמרים — קונה עוד OUI.

‎MAC = OUI (יצרן) + Serial (מספר כרטיס). מוטבע בכרטיס במפעל.
📘נקודות חשובות
  • כתובת ברודקאסט = FF:FF:FF:FF:FF:FF (כל 48 הביטים = 1). הודעה לכולם ב-LAN.
  • שתי כרטיסים יכולים בעיקרון להיות עם אותו MAC — כל עוד הם לא באותו LAN. כי MAC לוקאלי.
  • ‎MAC לא משתנה כשעוברים רשתות. ‎IP כן.

פרוטוקול ARP — Address Resolution Protocol

📘מה זה עושה

ARP = שירות שמקבל IP ומחזיר את ה-MAC המתאים, בתוך אותו LAN.

בכל כרטיס רשת יש ARP Table — טבלה שמחזיקה מיפויים של ‎IP → MAC לשכנים שהוא ראה.

איך ARP עובד — דוגמה

‎ARP — שאלה ב-broadcast, תשובה ב-unicast. בתוך LAN בלבד.
📘תהליך ARP מפורט

תרחיש: A רוצה לשלוח ל-B (10.0.0.2). A לא יודע את MAC של B.

  1. A בודק ARP Table. ריקה — אין מיפוי.
  2. A שולח ARP Query ב-broadcast: "מי בעל IP 10.0.0.2?".
    • ‎MAC dst = FF:FF:FF:FF:FF:FF (כולם)
    • ‎IP src/dst, MAC src של A
    • ‎Target IP = 10.0.0.2 (זה מי שאני מחפש)
  3. כולם ב-LAN מקבלים. רק B מזהה: "זה אני!".
  4. B שולח ARP Reply ב-unicast: "אני 10.0.0.2 וה-MAC שלי BB:..:02". שולח ישירות ל-A (לא broadcast).
  5. A מעדכן ARP Table: ‎10.0.0.2 ↔ BB:..:02.
  6. ‎ARP Table מתחזק — אבל הרשומה פגה אחרי כמה דקות (חברתי).

שליחת הודעה — בתוך LAN vs מחוץ ל-LAN

📘בתוך LAN (שכנים)

A שולח ל-B שכן ב-LAN:

  1. A יודע את ה-IP של B מהאפליקציה (DNS וכו').
  2. A בודק subnet mask — B אכן באותו subnet.
  3. A מקבל את ה-MAC של B דרך ARP.
  4. A יוצר פריים עם MAC dst = MAC של B. שולח. B מקבל. סוף.
📘מחוץ ל-LAN (דרך ראוטר)

A שולח ל-B שנמצא ב-LAN אחר:

  1. A יודע את ה-IP של B. בודק subnet — שונה ממנו.
  2. A מבין: צריך לעבור דרך ה-gateway (הראוטר).
  3. A מקבל את MAC של ה-gateway דרך ARP.
  4. A יוצר פריים עם MAC dst = MAC של הראוטר. אבל IP dst = IP של B! זה לא טעות — ה-IP נשאר עד היעד.
  5. הראוטר מקבל, מסיר פריים, רואה את ה-IP של B, מסתכל בטבלה, יוצר פריים חדש עם MAC dst = של הראוטר הבא או של B אם הוא בשכנות.
  6. וכן הלאה בכל hop.
עיקרון מרכזי: ה-IP נשמר מהשולח עד היעד. ה-MAC מתחלף בכל hop.

מה ההבדל בין כתובות MAC לכתובות IP?

‎4 הבדלים מרכזיים
  1. אורך: MAC = 48 ביט. ‎IP = 32 ביט (ב-IPv4).
  2. שכבה: MAC = שכבת הקישור (2). ‎IP = שכבת הרשת (3).
  3. היקף: MAC = לוקאלי, בתוך LAN. ‎IP = גלובלי, end-to-end.
  4. היררכיה: MAC = שטוח, ללא היררכיה (OUI לא מצביע על מיקום). ‎IP = יש subnets, היררכיה.
מתי משתמשים בכל אחד
  • MAC: בתוך LAN, להעברת פריים פיזית בין שני שכנים. מתחלף בכל hop.
  • IP: לניתוב מהמקור עד היעד, דרך כמה רשתות. נשאר זהה לאורך כל המסלול.
איך מקבלים את כל אחד
  • MAC: מוטבע בכרטיס במפעל. לא משתנה.
  • IP: מ-DHCP. משתנה כשעוברים רשת.
‎MAC = ת״ז פיזית (כתובת קישור-לוקאלית). ‎IP = כתובת מגורים (לוגית, גלובלית). שניהם נחוצים — IP לניתוב, MAC לזיהוי כרטיס במשלוח הפיזי.
📌סיכום הפרק
  • MAC = ‎48 ביט, מוטבע בכרטיס במפעל. ‎3 בתים OUI + 3 בתים serial
  • ‎Broadcast: FF:FF:FF:FF:FF:FF
  • ‎MAC משויך לכרטיס רשת, IP לרשת. ‎MAC קבוע, IP משתנה
  • ARP: ‎IP → MAC, בתוך LAN בלבד
  • ‎ARP Query = broadcast, ARP Reply = unicast
  • בתוך LAN: ‎A שולח ל-B עם MAC של B
  • מחוץ ל-LAN: ‎A שולח עם MAC של הראוטר. ‎MAC מתחלף בכל hop, IP נשאר
